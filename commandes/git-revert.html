git revert

La commande permet de rétablir des commits déjà éxistant. Elle sert donc à "remonter le temps" et annuler l'effet de certains commit précédemment fait En réalité, la commande ne supprime pas le commit choisit mais créé un commit automatique sans les modifications du commit choisit, à ne pas confondre avec git reset. Dans l'exemple, on fera un retour en arrière de 1 commit.

Syntaxe :

git revert

Affichage

git revert HEAD
git revert HEAD
[choisir le nom du commit]

git log
commit ...
Author: x
Date: YY
Le commit revert

git revert -e

Cette commande permet d'éditer le nouveau commit qui "supprimera" le commit choisit. La fonction -e ou --edit est l'option par defaut dans un terminal.

Syntaxe

git revert -e

Affichage

git revert -e HEAD
[choisir le nom du commit]

git log
commit ...
Author: x
Date: YY
Le commit revert

git revert -n

Normalement la commande git revert crée automatiquement un commit dont le message renseigne le(s) commit annulé(s). Ici, on annule effectivement les commit désirés dans le répertoire git, mais on ne crée pas de nouveau commit.

Syntaxe :

git revert -n

Affichage

commit b166a65027383c626bbfea7f8bbbf50e4b5d24f7
Author: Utilisateur
Date: Thu Mar 12 14:18:04 2015 +0100

Revert "Commit à revert"
