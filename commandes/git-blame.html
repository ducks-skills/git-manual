Cette commande va afficher les lignes d'un fichier et leur associer le dernier commit qui les concernes, afin de pouvoir comparer chaque ligne, et trouver d'eventuels problèmes.

Syntaxe :

$git blame nom_du_fichier
$ git blame exemple.html
a153f89d (Ylen96 2015-03-12 15:07:46 +0100 1) dossier X, Y et Z
1501d1d9 (Ylen96 2015-03-12 15:12:23 +0100 2) fichier exemple
a153f89d (Ylen96 2015-03-12 15:07:46 +0100 3)
1501d1d9 (Ylen96 2015-03-12 15:12:23 +0100 4) 0782592389

git blame -L

Description

Syntaxe :

$git blame-L l.début,l.fin nom_du_fichier
Exemple d'affichage dans le terminal

git blame --reverse

Au lieu de montrer la révision dans laquelle une ligne est apparu, la commande affiche la dernière version dans laquelle une ligne a existé. Ceci exige une gamme de révision comme le DÉBUT.. LA FIN où le chemin pour blame existe dans le DÉBUT.

Syntaxe :

git blame --reverse nom_du_fichier
dans le terminal
